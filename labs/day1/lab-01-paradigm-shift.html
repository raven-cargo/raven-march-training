Concept 1 of 4

Two Ways to Build Software

Before we can understand agentic AI, we need to see clearly what we're leaving behind — and why it's not enough anymore.

The Old Mental Model

Deterministic Code

You write every step. The computer executes exactly what you wrote. Nothing more.

Input
→
if/else
→
Output
  • ● Breaks when the world changes
  • ● Can't handle ambiguity
  • ● Every edge case must be written
  • ● Knowledge decays without maintenance
def review_pr(code): # ← You write every rule if "password" in code: return "fail" elif len(code) > 1000: return "too long" # 500 more rules...
The Agentic Mental Model

Goal-Directed Intelligence

You define the goal. The agent perceives, reasons, and acts — handling novelty as it comes.

Goal
→
Agent
→
Outcome
  • ● Adapts to novel situations
  • ● Handles ambiguity with judgment
  • ● Combines tools, knowledge, reasoning
  • ● Improves as context grows
# ← You describe the goal "Review this PR for security issues, style problems, and edge cases. Explain your reasoning for each finding in plain English." # Agent handles the rest ↑
The Key Insight

You're not replacing your engineering skills. You're redirecting them — from writing every step to designing goals, constraints, and context.

Concept 2 of 4

The PRAO Loop

Every agent action follows this four-phase cycle. Click each phase to understand what happens — and see a real Claude Code example.

STEP 1 OF 4

Perceive

The agent reads its environment: files, git history, test results, documentation, previous conversation. It builds a model of the current state before doing anything else.

Claude Code example: "I'll start by reading the codebase structure..." (reads package.json, src/index.ts, runs git log --oneline -10 to understand recent changes)
Remember: Agents loop through PRAO multiple times per task — not just once.

Your First 5 Commands

These are the commands you'll use most in the next 3 days — and probably for the next 3 years. Click any command to copy it.

Explore

Understand a codebase

Ask Claude to orient you in an unfamiliar project. Works on legacy code, other people's repos, or your own from 6 months ago.

claude "explain the architecture of this project"
Run from the project root directory
Create

Write tests for existing code

Claude reads the function, understands its contract, and generates tests covering happy path, edge cases, and error conditions.

claude "write comprehensive tests for UserService.ts"
Specify the file for precision
Review

Review your own changes

Claude reads your diff and reviews it like a senior engineer — checking for bugs, security issues, and style violations.

claude "review my changes in this PR for security issues"
Pair with git diff for best results
Create

Add documentation

Claude generates JSDoc, docstrings, README sections, or API documentation from code — never from thin air.

claude "add JSDoc comments to all exported functions in api.ts"
Specify the output format you want
Configure

Set up your project context

Create a CLAUDE.md file that tells Claude about your project — conventions, stack, rules, and team preferences. This persists across all conversations.

claude "create a CLAUDE.md for this project with our conventions"
Do this once per project
