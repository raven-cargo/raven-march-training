Your Capstone Track

The capstone is a production pipeline design for Raven Cargo. Each track applies the same five-phase framework (Problem Scoping → Architecture → Prompt Design → Safety → Validation) to a problem relevant to your role. Pick one and ship it.

Technical

Engineering & DevOps

For developers, engineers, and IT staff working on RavenEye or internal tooling.

T1
PR Review Pipeline
Simple Chain · ~4h

Build an agent that reviews every pull request to the RavenEye repository. The agent reads the diff + CLAUDE.md context, then posts a structured review comment covering security, performance, and test coverage.

Deliverables: (1) Trigger specification — PR opened webhook. (2) TCEF prompt for code review with Raven conventions. (3) settings.json with read-only scope — no write access to the codebase. (4) Output schema for the review comment with severity tags. (5) Tested on 3+ real PRs from the RavenEye repo.
T2
Incident Triage Router
Conditional Router · ~6h

Design a conditional router that classifies incoming alerts (tracking failures, API errors, carrier integration issues) and routes each to a specialized agent with the right MCP tools and system prompt.

Deliverables: (1) Classification logic for 3+ incident types. (2) Separate system prompts per specialist agent. (3) MCP server config for each specialist (Streamable HTTP transport). (4) Approval gate before any write action. (5) Incident response playbook covering the first 5 minutes.
T3
Data Quality Monitor
Parallel Fan-Out · ~5h

Deploy a scheduled pipeline that runs nightly to validate RavenEye data quality — one sub-agent per domain (shipments, carriers, invoices). Each checks records against schema rules and opens issues for violations above a severity threshold.

Deliverables: (1) Cron trigger specification. (2) Fan-out architecture with 3 domain sub-agents. (3) Validation rules per domain. (4) Cost budget: per-session ceiling + daily limit. (5) Alerting thresholds for error rate and cost spikes.
Accounting

Finance & Invoicing

For accounting, billing, and finance team members who handle carrier invoices, rate reconciliation, and cost reporting.

A1
Invoice Reconciliation Pipeline
Simple Chain · ~4h

Design an agent that compares carrier invoices against contracted rates and flags discrepancies. The agent reads the invoice data, cross-references the rate agreement, and produces a structured variance report with action items.

Deliverables: (1) Trigger: new invoice received or batch upload. (2) TCEF prompt specifying the reconciliation rules (lane, weight break, accessorials, fuel surcharge). (3) Output schema: variance report with line-item flags (over-charge, under-charge, missing accessorial). (4) Approval gate before any adjustment is posted. (5) Tested on 5+ sample invoices.
A2
Cost Allocation Report Generator
Simple Chain · ~3h

Build an agent that pulls shipment cost data from RavenEye, allocates costs by customer/lane/mode, and generates a formatted weekly cost report for leadership review.

Deliverables: (1) Scheduled trigger (weekly). (2) Data gathering prompt: which RavenEye fields to pull. (3) Allocation rules encoded as a skill file. (4) Output: formatted report with cost breakdown by customer, lane, and transport mode. (5) Permission scope: read-only access to financial data, no write.
A3
Rate Audit & Compliance Checker
Parallel Fan-Out · ~5h

Deploy an agent that audits active carrier rate agreements for expiration, missing accessorial definitions, and deviations from market benchmarks. One sub-agent per carrier, running in parallel.

Deliverables: (1) Fan-out architecture: one sub-agent per carrier. (2) Audit rules: expiration check, accessorial completeness, market deviation threshold. (3) Output: compliance scorecard per carrier with red/yellow/green flags. (4) Cost controls: per-session token ceiling. (5) Quarterly maintenance cadence documented.
Business

Operations & Leadership

For operations managers, dispatchers, account managers, and leadership who manage carrier relationships, customer communication, and strategic decisions.

B1
Carrier Performance Dashboard Generator
Simple Chain · ~3h

Build an agent that pulls carrier performance data (on-time %, claims rate, capacity utilization) and generates a weekly executive summary with trend analysis and recommendations for carrier mix adjustments.

Deliverables: (1) Scheduled trigger (weekly, Monday morning). (2) Data gathering from RavenEye carrier metrics. (3) Output: executive summary with top 5 carriers, bottom 5 carriers, and trend arrows. (4) Approval gate before any summary is distributed externally. (5) Brand guidelines embedded in the agent persona file.
B2
Customer Exception Communication
Conditional Router · ~4h

Design an agent that monitors shipment exceptions (delays, damage, missed pickups) and drafts customer-facing communications routed by severity — routine status updates for minor delays, escalation notices for critical issues.

Deliverables: (1) Exception classification rules (minor/moderate/critical). (2) Communication templates per severity level. (3) Conditional routing: minor → auto-draft, critical → human approval gate. (4) Tone and brand guidelines encoded as a skill. (5) Tested on 5+ real exception scenarios from Raven's history.
B3
Customs & Cross-Border Documentation
Simple Chain with Approval Gate · ~5h

Build an agent that assembles cross-border documentation packages (BOL, commercial invoice, packing list, customs declarations) from shipment data, validates completeness, and presents the package for broker review before submission.

Deliverables: (1) Trigger: shipment flagged as cross-border in RavenEye. (2) Document assembly rules encoded as a W.A.T. framework (Workflows, Agent, Tools). (3) Completeness validation checklist. (4) Approval gate: broker reviews before submission — agent never submits autonomously. (5) Blast radius documented: maximum scope of damage from a documentation error.

The Five Design Phases

Every capstone track uses the same five-phase framework. Complete each phase in order before writing any configuration.

1
Problem Scoping Define the trigger, write a measurable success criterion, and trace the value chain from output to beneficiary.
2
Pipeline Architecture Choose your archetype (simple chain, fan-out, or conditional router). Document whether single-agent or multi-agent, and why.
3
Prompt & Skill Design Write the full TCEF prompt. Identify patterns to encode as skill files. Define the output schema.
4
Safety & Permissions Write settings.json allow/deny lists. Audit secrets. Place approval gates for irreversible actions. Document blast radius.
5
Validation Test on 5+ representative real inputs. Validate output against schema. Test edge cases and approval gates.
