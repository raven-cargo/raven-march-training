Task Suitability Framework · Module 1 / M01

AI Task Suitability Matrix

Freight operations classified by agent fitness — the core mental model from Day 1, Block 1.

Best Fit for AI

  • Extract shipment details from carrier email
  • Classify carrier communications by urgency
  • Summarize exception reports from last week
  • Extract BOL fields (shipper, consignee, PRO#, SCAC)
  • Score carriers by FMCSA + claims + on-time %
  • Parse rate quotes from carrier rate sheets
  • Check carrier authority on FMCSA SAFER
  • Monitor tracking feeds for delay signals
  • Generate exception reports for delayed loads
  • Verify insurance expiration across carrier pool
  • PR review against RavenEye codebase patterns
  • Morning standup: summarize overnight exceptions

Needs Better Spec

  • Decide whether to rebook a delayed shipment
  • Evaluate carrier fitness for new lane
  • Predict optimal shipment consolidation
  • Suggest contingency routing for weather events
  • Auto-assign loads to carrier pool
  • Assess hazmat documentation completeness

Poor Fit (Human Required)

  • Negotiate a new lane rate with a carrier
  • Resolve customer disputes over claims
  • Make strategic capacity commitments
  • Approve large financial transactions
  • Handle sensitive compliance escalations
  • Build new carrier relationships

Model Context Protocol · Freight Operations

MCP Architecture for Raven Cargo

The three MCP primitives mapped to Raven's freight operations — "USB-C for AI capabilities."

Tools (Side Effects)

  • Write to RavenEye API — update shipment status, carrier records
  • Update carrier records — scoring, authority status changes
  • Trigger notifications — delay alerts, ETA updates to customers
  • Check carrier authority — FMCSA SAFER lookup
  • Submit BOL extraction — process document and store results
  • Rebook shipment — assign to new carrier (HUMAN GATE)

Resources (Read-Only)

  • Carrier database — profiles, lanes, rates
  • FMCSA SAFER data — authority, insurance, safety
  • Live tracking feeds — GPS position of active shipments
  • Claims history — past claims and resolution data
  • TMS data — shipment records, booking history
  • RavenEye dashboard — aggregated operational metrics

Prompts (Templates)

  • "Review this BOL" — extract fields with confidence scores
  • "Score this carrier" — multi-criteria evaluation
  • "Exception report" — delayed loads for date range
  • "Carrier compliance check" — full verification
  • "Morning standup brief" — overnight exception summary
  • "Rate comparison" — compare quotes across carriers

Operational Skills · Slash Commands

Raven Cargo Slash Commands

Reusable agent commands built from freight operational skills — the persistence layer that solves the stateless session problem.

/carrier-check [SCAC]
Check carrier operating authority, active insurance, and safety rating. Cross-references FMCSA SAFER with internal carrier database.
Output: { authority, insurance_status, safety_rating, last_inspection }
/bol-extract [file]
Structured extraction pipeline for BOL documents. Extracts shipper, consignee, commodity, weight, PRO#, SCAC with confidence scores.
Output: { fields: [...], confidence: 0.0-1.0, warnings: [...] }
/exception-report [date-range]
Summarize operational exceptions — delays, missed pickups, claims, detention — for the specified date range.
Output: { summary, exceptions: [...], trends, action_items }
/carrier-score [MC#]
Multi-criteria carrier scoring: FMCSA authority, claims history, on-time %, lane performance, insurance coverage.
Output: { overall_score, criteria: {...}, recommendation }
/ship-check [PRO#]
Real-time shipment status via RavenEye. Returns current location, ETA, exceptions, and carrier contact.
Output: { location, eta, status, carrier, exceptions }
/rate-compare [lane]
Compare carrier rate quotes for a specific lane. Pulls from rate sheets and historical TMS data.
Output: { carriers: [...], recommended, savings_pct }
Skill YAML Template · Lab 06, CE-06-C

Skill Structure

---
name: carrier-scorer
description: Score carriers using FMCSA data, claims history, and lane performance
role: >
  You are a carrier intelligence analyst for Raven Cargo. You evaluate
  motor carriers using federal safety data, claims records, and operational
  performance metrics to produce actionable scoring reports.
procedure:
  1. Retrieve carrier profile from FMCSA SAFER using MC or USDOT number
  2. Verify operating authority is ACTIVE and insurance is current
  3. Pull claims history from internal carrier database
  4. Calculate on-time delivery percentage from TMS records
  5. Evaluate lane-specific performance (if lane specified)
  6. Produce structured JSON score with criteria breakdown
anti_patterns:
  - Never score a carrier without verifying active authority first
  - Never hallucinate safety ratings — use only FMCSA data
  - Never recommend a carrier with expired insurance
---
