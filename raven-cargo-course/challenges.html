Raven Concept Challenges

Seven challenges derived from the video concepts above, rewritten for Raven Cargo operations. Use the tabs to filter by role.

Non-Technical

Operations & Business Challenges

C1
The Freight Rate Challenge
Foundational · “What” Over “How” Philosophy

Agentic workflows shift from step-by-step automation to goal-oriented execution. You provide the desired end goal and the agent figures out the logic, API calls, and tools required.

Challenge: Write a plain-language objective telling your agent to pull current LTL and FTL rate quotes from three carriers Raven Cargo works with and consolidate them into a single comparison spreadsheet. Provide no instructions on column formatting, rate normalization, or which RavenEye API endpoints to use. Evaluate whether the agent autonomously infers the required fields (lane, weight break, transit days, accessorials) or asks clarifying questions.
C2
The Tracking Dictation Challenge
Foundational · Hands-Free Voice Orchestration

Leverage OS-level dictation tools to bypass the keyboard bottleneck. Speak your goals naturally and the software translates voice into executable agentic code.

Challenge: Without touching your keyboard, use a voice-to-text tool to verbally describe an automation that takes a Raven PRO number as input, queries the RavenEye tracking feed for the latest shipment location and status, and drafts a customer update email with the current ETA and any exception flags. Execute the workflow to confirm the agent captured your spoken intent accurately.
C3
The Customs SOP Challenge
Business Operations · W.A.T. Framework

The W.A.T. framework organizes agentic tasks into Workflows (natural-language SOPs), the Agent (the AI coordinator), and Tools (the executable scripts).

Challenge: Create a CLAUDE.md file that acts as your Raven operations onboarding document using the W.A.T. framework. Instruct your agent to read it and autonomously generate the directory structure (Workflows, Tools, Temp) tailored for Raven Cargo's cross-border customs documentation process — including BOL generation, customs broker handoff templates, and carrier compliance checklists.
C4
The Fleet Analytics Challenge
Business Operations · Multi-Agent Corporate Personas

Build specialized teams of agents representing distinct departments, equipped with specific skills and MCP tool integrations to pull live data.

Challenge: Define an agent persona file assigning the role of ‘Raven Operations Analyst’ with embedded Raven Cargo brand guidelines (colors, logo, report format). Feed the agent a raw dataset of Raven's monthly carrier performance metrics (on-time %, claims rate, lane coverage, cost-per-mile) and command it to independently generate a 10-slide executive presentation for leadership that follows Raven's brand constraints.
Technical

Engineering & DevOps Challenges

C5
The Waybill Flow Control Challenge
Technical · Agentic Prompt Composability

High-level agentic prompts must be modular, utilizing dynamic variables and Control Flow logic for loops or conditional early returns without manual intervention.

Challenge: Engineer a composable prompt with a dynamic {{PRO_NUMBER}} variable that triggers a Raven shipment status lookup. Include explicit control flow: if no PRO number is provided, halt execution and ask the dispatcher to supply it; if the shipment shows an exception status in RavenEye, branch into an escalation sub-prompt that drafts a carrier communication and flags the load for ops review.
C6
The Routing Algorithm Concurrency Challenge
Technical · Parallel Orchestration

Orchestrate multiple agents in parallel to prevent context bloat and linear execution times. Use Git Worktrees or shared task lists for sub-agents to collaborate.

Challenge: Implement a slash command that spins up three parallel sub-agents against the RavenEye codebase. Agent 1 audits the carrier API integration layer for security vulnerabilities and credential exposure. Agent 2 profiles the shipment tracking query performance and identifies latency bottlenecks. Agent 3 evaluates test coverage across the booking and dispatch modules. The coordinator agent aggregates all three reports into a single prioritized executive summary for the engineering lead.
C7
The RavenEye CI/CD Challenge
Advanced · Headless Infrastructure & CI/CD

Deploy agents in Headless Mode on scalable cloud infrastructure. Run autonomously 24/7, triggered by webhooks, cron jobs, or new code pushes.

Challenge: Write a GitHub Actions workflow YAML that deploys a headless Claude Code agent triggered on every pull request to the RavenEye repository. The agent's system prompt must audit PRs specifically for: (1) changes touching carrier rate tables or financial calculations, (2) modifications to the shipment tracking API surface, and (3) any new environment variable usage that could leak credentials. Output a structured review comment with severity-tagged findings.
