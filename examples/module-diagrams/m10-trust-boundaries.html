Trust boundaries and tool gates

Security flow Prompt injection defense is not one trick. It is a chain of explicit boundaries: sanitize the artifact, frame it as data, validate the model output, and gate any tool call that could create side effects.
▶ Click each boundary to see what it defends, what signal it emits, and what failure looks like if the layer is missing.
Ingest
Delimit untrusted content before the model reads it.
Frame
Tell the model where trusted instructions come from.
Validate
Reject output that breaks the contract.
Gate
Authorize tool execution with policy and logs.
▶ A hardened path looks like this. If one node is missing, the next node inherits unnecessary ambiguity.
Source
File, page, ticket, DB row
Boundary

Model
