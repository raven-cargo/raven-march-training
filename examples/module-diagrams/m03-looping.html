Looping vs thoroughness

Key judgment call Both looping and thoroughness involve the agent reading multiple files. The difference determines whether you let it work or step in. This is one of the most common misjudgments engineers make when watching an agent.
▶ Compare these two traces side-by-side. Same number of reads — very different behavior.
Thoroughness
Read("src/auth/jwt.ts") thinking: JWT module found. Uses HS256. → discovered: imports from ./config Read("src/auth/config.ts") ← new file thinking: config exports JWT_SECRET and EXPIRY. → discovered: references userService Read("src/services/userService.ts") ← new file thinking: userService uses jwt.verify in getUser. → discovered: test file exists Read("src/auth/jwt.test.ts") ← new file thinking: tests cover sign/verify but not refresh. Now I have the full picture. Planning changes...
Each read is a new file discovered from the previous one. Thinking covers new ground each time. The agent is building a coherent model. Let it work.
Looping
Read("src/auth/jwt.ts") thinking: could use approach A (RS256)... → doesn't resolve, considers approach B Read("src/auth/jwt.ts") ← same file thinking: approach B would work, but what about backward compatibility? Maybe approach A... Read("src/auth/jwt.ts") ← same file again thinking: if I use approach A, the config issue... → same reasoning, no new conclusion Read("src/auth/jwt.ts") ← same file again thinking: not sure whether A or B. Perhaps...
Same file read repeatedly. Thinking rehearses the same options without reaching a conclusion. No new information being gathered. The agent is stuck — intervene with a decision.
The diagnostic rules
1
Are the reads hitting new files? New files = thoroughness. Same file without intervening writes = likely loop.
2
Is the thinking covering new ground? New observations, new files discovered, new constraints identified = thoroughness. Same options reconsidered = loop.
3
Has there been a write between repeated reads? Re-reading after a write makes sense — the content changed. Re-reading without a write = the agent is looking for something it already found.
4
Your intervention for a loop: Answer the question the agent is circling. "Use RS256, store refresh tokens in PostgreSQL, maintain backward compatibility for existing tokens." Specific decisions break loops.
