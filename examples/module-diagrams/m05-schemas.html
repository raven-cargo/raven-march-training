Reading tool schemas

Schema literacy Tool schemas are the contract between an MCP server and any client. Claude reads them at connection time to decide when and how to invoke tools. Understanding the anatomy of a schema — especially the description field — is essential for writing and evaluating MCP servers.

Writing descriptions for Claude, not humans
For humans (insufficient)
"Searches issues in the project tracker."
For Claude (includes 3 required elements)
"Search for issues matching criteria. Use when finding existing issues by keyword/status/assignee. Do NOT use to create issues (use create_issue) or get by ID (use get_issue). Returns array of matches. Empty array if no matches."
1
The action: What the tool does in precise terms. Not "searches issues" — "searches for issues matching given criteria."
2
When to use it: Conditions under which this tool is right, and when to use alternatives instead. This prevents Claude calling the wrong tool.
3
What it returns: Structure and content of the output so Claude can use the result correctly in its next reasoning step.
