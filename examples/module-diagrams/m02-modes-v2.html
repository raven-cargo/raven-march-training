Two modes of Claude Code

Learning objective Choose the right invocation mode for any task. Interactive mode keeps you in the loop; non-interactive mode produces pipeable output for automation. Knowing which to reach for saves real time.
▶ Switch between tabs to compare the two modes side-by-side.
$ claudeInteractive mode
$ claude -p "..."Non-interactive mode
Context
Accumulates across exchanges — each message sees full history
Human role
In the loop — can redirect, answer questions, add context mid-task
Session end
History vanishes. Decisions must be written to CLAUDE.md or files.
Output
Streamed to terminal with thinking, tool calls, and responses
Strong fit
●Exploratory work — you're learning the task as you go
●Complex multi-step refactoring where you might redirect
●Debugging — each discovery changes direction
●Initial project setup with clarifying questions
# Start a session, work interactively $ claude > Refactor auth to use JWT. Keep tests passing. # ...watch it work, redirect if needed... > Actually, use RS256 not HS256 for the signing. # ...review output, then persist decisions... > Add the JWT decision to CLAUDE.md under Decided.
Context
Single-shot — reads CLAUDE.md, executes prompt, exits. No history.
Human role
Not in the loop — you define the task upfront, review output after
Session end
Immediate — prints result to stdout and process terminates
Output
Stdout only — pipeable, redirectable, capturable in variables
Strong fit
●CI/CD pipelines — automated code review on PRs
●Scripted workflows — chaining agent tasks in shell
●Scheduled automation — nightly docs or test generation
●Integration — agent output as input to another tool
# Single-shot: pipe diff into review, email result REVIEW=$(claude -p "Review for security issues: $(git diff main)") echo "$REVIEW" | mail -s "Security Review" team@co.com # Generate changelog from git log claude -p "Generate changelog: $(git log $(git describe --tags --abbrev=0)..HEAD --oneline)" # Check JSDoc coverage claude -p "Check if all exports in src/api.ts have JSDoc comments"
